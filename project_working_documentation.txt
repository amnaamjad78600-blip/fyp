1. PROJECT OVERVIEW & VALUE PROPOSITION
--------------------------------------------------------------------------------
The AI Fashion Stylist App is a culturally-tailored personal style consultant 
designed for Pakistani and South Asian clothing cultures. Traditional general-purpose
styling apps fail to understand local cultural wear, regional brand contexts, and local
currencies. This app bridges that gap by offering:
- Multi-modal visual style analysis using user-uploaded portrait photos.
- Precise detection of skin tone, body shape, and personal characteristics.
- Localized recommendations featuring actual Pakistani brands (e.g., Khaadi, 
  Maria B, Sapphire, J., Alkaram) priced in PKR.
- A dialogic, conversational AI Stylist Chat to ask for styling and matching tips.
- Offline-first dynamic synchronization ensuring image files and analysis inputs 
  are backed up locally as well as on Cloud Firestore.

3. BACKEND & SERVICE-LEVEL WORKING
--------------------------------------------------------------------------------

The backend operates on a flat, high-performance root-level architecture in Cloud 
Firestore. All dynamic data is isolated by user tags rather than nested subcollections.

📂 [lib/services/auth_services.dart](file:///c:/Users/amnaa/Documents/ai_fashion_app/lib/services/auth_services.dart)
- Purpose: Authenticates users and enforces database credentials sync.
- Working:
  * Manages Sign In, Sign Up, and Sign Out operations.
  * **Self-Healing Profile Creation**: Whenever a user logs in or signs up, it checks
    if their root `users/$uid` document is present in Firestore. If it is missing, it
    automatically heals the backend by re-creating a secure, pristine user profile 
    document containing only `email`, `username`, and `profileImage`.

📂 [lib/services/firestore_service.dart](file:///c:/Users/amnaa/Documents/ai_fashion_app/lib/services/firestore_service.dart)
- Purpose: Global CRUD coordinator for Cloud Firestore.
- Working:
  * Coordinates all database operations on root collections:
    * `users` - Keeps pristine profile details.
    * `favorites` - Keyed by `${uid}_$slug` for quick, collision-free reads.
    * `uploaded_images` - Stores uploaded base64 photos and local backups.
    * `ai_recommended_images` - Houses dynamic outfit suggestions with generated links.
    * `chat_history` - Stores user-stylist conversations.
    * `preferences` - Keeps track of design aesthetics.
    * `last_analyses` - Captures skin tone, body type, and personal parameters.

📂 [lib/services/storage_service.dart](file:///c:/Users/amnaa/Documents/ai_fashion_app/lib/services/storage_service.dart)
- Purpose: Hybrid storage coordinator for media assets.
- Working:
  * **Cloud Storage**: Uploads styling photos and profile pictures directly to Firebase 
    Storage buckets, generating public download URLs.
  * **Local Storage Backup**: Saves high-resolution .jpg and .png backups on the 
    device's local storage directory, allowing the app to stay functional and load
    images extremely fast even with poor network connectivity.

4. AI MODEL INTEGRATION & ARCHITECTURE DETAILS
--------------------------------------------------------------------------------

### 🤖 Model Utilized in the Project
The primary engine powering the intelligent matching logic is:
👉 **`meta-llama/llama-4-scout-17b-16e-instruct`** (Run via high-frequency **Groq API Cloud Infrastructure**).

### ⚙️ How the Vision & Stylist AI Engine Works
1. **Multi-Modal Feature Scan**:
   * The user uploads their face or styling image.
   * The image is converted into a base64 string on the frontend.
   * It is sent alongside a custom styling prompt directly to the Llama Vision-Language 
     Model (VLM) API.
2. **Skin Tone & Gender Classification**:
   * The model scans the visual features of the uploaded face to determine their skin
     tone (Fair, Wheatish, Medium, Dark) and apparent gender.
3. **Prompt Conformance & Regional Adaptations**:
   * The model combines these visual details with the user's text search query (e.g. 
     "traditional wedding lawn").
   * It constructs a curated catalog of exactly 10 outfit suggestions.
   * It enforces strict Pakistani fashion context: outputting local brands, pricing in 
     PKR, and traditional garments like Lehenga, Gharara, Sherwani, and Shalwar Kameez.
4. **Structured JSON Output Constraints**:
   * Enforces a structural JSON schema format, returning skin, body, season, and list 
     objects that match the query exactly.
5. **Smart Live Internet Image Scraper fallback**:
   * Once the Llama Model returns the JSON suggestions, the backend processes them.
   * If there are no images, the app dynamically queries **Unsplash's Search Engine**
     for the exact outfit description (e.g., "Khaadi Kurta") using regex parsers to 
     extract real-time, matching high-definition stock photos on the fly!

5. FUTURE ADVANCEMENTS & SCALABILITY PATHWAYS
--------------------------------------------------------------------------------

To take this application from a startup prototype to a world-class commercial product,
the following advancements can be implemented in future phases:

### 🌟 Future AI Model Advancements
1. **Virtual Try-On (VTO) Integration**:
   * Future improvement: Integrate diffusion models (like Stable Diffusion XL or 
     TryOnDiffusion models).
   * How it works: By taking the user's uploaded body photo and the recommended outfit's 
     image, the model can virtually drape the clothes onto the user's body, letting 
     them see how the dress fits before purchasing.
2. **On-Device Offline LLMs**:
   * Future improvement: Deploy lightweight, on-device open-source LLMs (like 
     Google Gemma-2B or Llama-3.2-3B) using Google Mediapipe or Ollama on mobile.
   * Benefit: Allows the conversational stylist chat to operate 100% offline without 
     relying on internet connectivity or Groq API calls!
3. **Visual Similarity Search**:
   * Future improvement: Integrate vector search databases (like Pinecone or Firebase 
     vector indexing).
   * Benefit: Users can take a photo of any outfit they see in public, and the app 
     will find matching styles from Pakistani brands instantly.

### 🌟 Future Project Improvements
1. **Direct Brand Affiliate Checkout**:
   * Connect the recommended outfits to live e-commerce stores. Add "Buy Now" links that 
     take users directly to Khaadi, Maria B, or Sapphire checkouts with pre-loaded items.
2. **Size Recommendation Engine**:
   * By taking custom measurement inputs (chest, height, shoulders), compare their data 
     with standard brand charts to recommend the exact perfect size (S, M, L, XL).
3. **Social Fashion Feed**:
   * Enable users to share their matched recommendations on a public, styled feed 
     allowing other South Asian users to get dress ideas and inspirations.
